## 一、本节学习目标

- 理解 YAML 配置管理的核心思想
- 掌握 `pyyaml` 库的基本用法
- 能够封装一个统一的配置加载器
- 熟悉本项目 4 个核心配置文件的用途

## 二、核心知识点讲解

### 2.1 为什么需要配置管理？
- **解耦**：把"参数"和"代码逻辑"分离，改参数不用动代码
- **多环境**：开发/测试/生产环境用不同配置
- **集中管理**：所有路径、参数集中在一处，便于维护

### 2.2 YAML 基础语法
```yaml
# key: value 形式
chat_model_name: qwen-turbo
timeout: 10

# 嵌套对象
ticket_client:
  base_url: ""
  timeout: 10
  retry: 2

# 列表
allow_knowledge_file_type: ["txt","pdf"]
```

### 2.3 本项目 4 个核心配置
| 配置文件 | 职责 |
|----------|------|
| `config/rag.yml` | 大模型/Embedding 名称 |
| `config/chroma.yml` | 向量库参数（切片大小、Top-K） |
| `config/prompts.yml` | Prompt 文件路径映射 |
| `config/agent.yml` | 数据路径、票务客户端配置 |

## 三、项目落地场景

项目启动时，配置加载器会被**各模块自动调用**：
- 模型层：读取 `rag.yml` 选择用哪个模型
- RAG 层：读取 `chroma.yml` 决定如何切分文档
- Prompt 层：读取 `prompts.yml` 找到模板文件路径
- Agent 层：读取 `agent.yml` 知道数据在哪

## 四、关键代码+逐行注释

### 4.1 路径工具 utils/path_tool.py
```python
"""
为整个工程提供统一的绝对路径
无论你在哪里运行脚本，路径都指向项目根目录
"""
import os

def get_project_root() -> str:
    """
    获取工程所在的根目录
    :return: 字符串根目录
    """
    # 当前文件的绝对路径（utils/path_tool.py 的绝对路径）
    current_file = os.path.abspath(__file__)
    # 获取文件所在文件夹的绝对路径
    current_dir = os.path.dirname(current_file)
    # 再向上跳一级，得到工程根目录
    project_root = os.path.dirname(current_dir)

    return project_root

def get_abs_path(relative_path: str) -> str:
    """
    传递相对路径，得到绝对路径
    :param relative_path: 相对于工程根的路径
    :return: 绝对路径
    """
    project_root = get_project_root()
    return os.path.join(project_root, relative_path)

if __name__ == '__main__':
    print(get_abs_path("config/config.txt"))
```

### 4.2 配置加载器 utils/config_handler.py
```python
"""
统一加载所有 YAML 配置文件
"""

import yaml
from utils.path_tool import get_abs_path

def load_rag_config(config_path: str = get_abs_path("config/rag.yml"), encoding: str = "utf-8"):
    """加载 RAG 配置（模型名）"""
    with open(config_path, "r", encoding=encoding) as f:
        return yaml.load(f, Loader=yaml.FullLoader)

def load_chroma_config(config_path: str = get_abs_path("config/chroma.yml"), encoding: str = "utf-8"):
    """加载 Chroma 配置（向量库参数）"""
    with open(config_path, "r", encoding=encoding) as f:
        return yaml.load(f, Loader=yaml.FullLoader)

def load_prompts_config(config_path: str = get_abs_path("config/prompts.yml"), encoding: str = "utf-8"):
    """加载 Prompt 路径配置"""
    with open(config_path, "r", encoding=encoding) as f:
        return yaml.load(f, Loader=yaml.FullLoader)

def load_agent_config(config_path: str = get_abs_path("config/agent.yml"), encoding: str = "utf-8"):
    """加载 Agent 配置（数据路径、客户端）"""
    with open(config_path, "r", encoding=encoding) as f:
        return yaml.load(f, Loader=yaml.FullLoader)

# 模块加载即缓存，全局共享
rag_conf = load_rag_config()
chroma_conf = load_chroma_config()
agent_conf = load_agent_config()
prompts_conf = load_prompts_config()

if __name__ == '__main__':
    print(agent_conf["chat_model_name"])
```

## 五、验收标准+踩坑避坑点

### 5.1 验收标准
- [ ] 4 个 YAML 文件均能独立 `yaml.load` 成功
- [ ] `path_tool.get_abs_path("config/rag.yml")` 能返回正确绝对路径
- [ ] 启动 `python utils/config_handler.py` 能打印 `qwen-turbo`
- [ ] 修改 `chroma.yml` 的 `k: 3` 为 `k: 5`，全局生效

### 5.2 踩坑避坑点
1. **路径分隔符**：Windows 下 `os.path.join` 会自动处理反斜杠，不要手写
2. **YAML 编码**：配置文件必须是 UTF-8，否则中文注释会乱码
3. **Loader 选择**：必须用 `yaml.FullLoader`，否则会有安全警告
4. **相对路径陷阱**：永远不要在业务代码里用相对路径，必须走 `get_abs_path`
5. **重复加载问题**：配置作为模块级单例 `rag_conf = ...`，避免每次调用都重新读盘
