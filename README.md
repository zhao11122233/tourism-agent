# 文旅智能助手 — 查票 · 订票 · 优惠 · 路线 · 导览

## 项目概述

基于 **LangGraph** 状态管理 + ChromaDB 向量知识库 + RAG 检索增强生成技术，构建面向 **C 端游客** 的文旅全链路智能体。覆盖 **需求咨询 → 查票 → 订票 → 优惠判定 → 价格计算 → 路线规划 → 交互式导览** 完整业务闭环。

独立客户端层封装票务 API，Agent 层专注对话交互与意图路由，两层严格分离。

核心亮点：
- **两阶段政策咨询**：无年龄/日期时先展示景区收费方案并主动邀请，有完整信息后精准计算每人明细
- **年龄→优惠档位自动映射**：识别"4岁""70岁"后自动细化为"儿童_6岁以下""老人_70岁以上"，直击免票/半价规则
- **淡旺季价格自动切换**：根据游玩日期匹配景区旺季/淡季基准票价

## 核心特性

#### 1. LangGraph 意图路由
- 会话状态管理（景区、日期、人数、手机号），自动提取上下文信息
- 意图识别 + 条件路由：查票 / 订票 / 优惠政策 / 路线规划 / 景点导览
- 信息缺项时逐个追问，已获取信息不重复索要

#### 2. 查票与订票
- `ticket_query`：查询美团 / 携程 / 景区官方多平台余票与票价
- `ticket_book`：提交门票预订，手机号校验，返回订单确认
- 独立 `client/ticket_client.py` 封装网络请求，Agent 层零网络依赖

#### 3. 特殊人群优惠智能判定
- 老人（60-69 半价 / 70+ 免票）、儿童（6 岁以下免票 / 6-18 半价）、军人（现役/退伍 免票）、残疾人（1-4 级免票 + 陪护优惠）、聋哑人士（免票）
- `parse_user_info` 自动提取年龄并映射到精确优惠档位
- `verify_discount` 逐一判定每人优惠等级，列出所需证件

#### 4. 统一价格数据源
- 景区价格、淡旺季时段、平台服务费统一存放在 `data/pricing/scenic_spots.json`
- `agent_tools.py` 和 `ticket_client.py` 通过模块级缓存共享同一数据源
- 修改价格只需改 JSON 文件，无需动代码

#### 5. RAG 知识库检索
- 5 大文旅知识领域：票务价格规则、特殊群体优惠政策、入园凭证核验规则、景区导览讲解材料、短信验证码格式
- ChromaDB 向量存储，Top-3 召回，LLM 总结生成回答

#### 6. 交互式景点导览
- 80+ 景区细分点位（故宫午门 / 长城北一楼 / 黄山迎客松等）
- 自动点位→景区关联，内置讲解词 + RAG 知识库回退

#### 7. Streamlit 流式对话
- 上下文记忆 + 场景化追问 + 参数格式校验 + 手机号隐私保护

## 8 个工具总览

| # | 工具 | 功能 | 入参亮点 |
|---|------|------|----------|
| 1 | `parse_user_info` | 解析人数、人员类型、年龄、核心诉求 | 提取年龄后自动映射到优惠档位 |
| 2 | `search_policy` | RAG 检索优惠规则、票务政策、导览材料 | 向量库召回 |
| 3 | `verify_discount` | 判定优惠等级（免票/半价/全价）+ 所需证件 | 三层容错解析 + 模糊匹配 |
| 4 | `calc_ticket_price` | 查基准票价（含淡旺季）→ 应用折扣 → 计算总价 | `visit_date` 参数驱动淡旺季切换 |
| 5 | `ticket_query` | 查询多平台余票和票价 | 美团/携程/景区官方三平台 |
| 6 | `ticket_book` | 提交门票预订，手机号校验，返回订单确认 | 校验下沉至客户端层 |
| 7 | `plan_route` | 生成分段式游览路线（节点/时长/无障碍） | 老人慢速 1.6× 调整 |
| 8 | `guide_order_exec` | 景点讲解 / 互动答疑 / 预订单生成 | 80+ 细分点位自动关联景区 |

## 分层架构

```
┌─────────────────────────────────┐
│  Streamlit UI (app.py)          │  ← 对话界面
├─────────────────────────────────┤
│  LangGraph Agent                │  ← 意图路由、状态管理、两阶段政策咨询
│  (agent/travel_agent.py)        │
├─────────────────────────────────┤
│  8 Tools (agent_tools.py)       │  ← 工具调用、年龄细化、淡旺季计算
├─────────────────────────────────┤
│  TicketClient (client/)         │  ← HTTP 请求、超时、重试、异常兜底
├─────────────────────────────────┤
│  ChromaDB RAG + qwen-turbo LLM  │  ← 知识检索 + 生成
└─────────────────────────────────┘
```

Agent 层 **不处理** 网络请求、接口鉴权、签名、密钥 — 全部在 TicketClient 层隔离。

## 政策咨询两阶段模式

```
阶段 1（无年龄/日期）："故宫门票多少钱"
  → search_policy 检索景区收费方案
  → 展示价格 + 邀请："需要帮您精准计算吗？请告知人数、年龄、身份、日期"
  → 用户拒绝 → 友好结束

阶段 2（有年龄/日期）："两个70岁老人6月15号去故宫"
  → parse_user_info 提取年龄 → 细化为"老人_70岁以上"
  → 检查信息完整性 → 不完整则追问
  → search_policy → verify_discount → calc_ticket_price
  → 展示：每人明细 + 附加项目提醒 + 总价 + AI 免责声明
```

## 快速开始

### 环境要求
- Python 3.10 及以上
- 阿里云百炼 DashScope API Key（免费额度足够日常使用）

```bash
git clone https://github.com/zhao11122233/tourism-agent.git
cd tourism-agent
pip install -r requirements.txt

# 配置 API Key
echo DASHSCOPE_API_KEY=your-api-key > .env

# 构建向量知识库
python -m rag.vector_store

# 启动
python -m streamlit run app.py
```

## 后续优化方向

> 以下为规划中的能力扩展，暂未实现。

- 集成手语识别与合成能力，为聋哑游客提供无障碍导览体验

#### 📸 多模态交互
- **语音**：语音输入提问 + TTS 语音播报导览讲解，解放双手
- **图片**：拍照识别景点 / 门票 / 指示牌，OCR 提取文字信息

- **政策变更感知**：知识库接入景区官网/文旅局公告，票价调整、优惠政策变动自动更新
- **维护通知**：景区某区域临时关闭/施工维护信息实时入库，路线规划自动规避

#### 🖥️ 知识库管理后台

- **内容审核**：上传后自动分块→向量化→入 ChromaDB，支持预览和手动编辑分块内容
- **版本管理**：知识条目变更历史可追溯，误操作可回滚到历史版本

#### 🤖 多 Agent 全行程规划
- **酒店 Agent**：根据目的地、预算、偏好自动推荐住宿方案
- **航班 Agent**：整合机票、高铁、景区摆渡车等多模式交通规划
- **活动 Agent**：基于目的地推荐当地演出、节庆、美食等体验活动
