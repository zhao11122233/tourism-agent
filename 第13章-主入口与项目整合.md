## 二、核心知识点讲解

### 2.1 Streamlit 聊天三件套

```python
# 1) 显示历史消息
for message in st.session_state["message"]:
    st.chat_message(message["role"]).write(message["content"])

# 2) 输入框
prompt = st.chat_input()

# 3) 流式输出
st.chat_message("assistant").write_stream(generator)
```

- `st.chat_message`：渲染聊天气泡
- `st.chat_input`：固定在底部的输入框
- `write_stream`：接收生成器，逐字渲染

### 2.2 session_state 管理

```python
if "agent" not in st.session_state:
    st.session_state["agent"] = TravelAgent()  # 只初始化一次

if "message" not in st.session_state:
    st.session_state["message"] = []  # 对话历史
```

Streamlit 每次交互都**重新执行整个脚本**，`st.session_state` 是唯一跨交互持久化的存储。把 Agent 实例和消息列表放进去，避免每次重建。

### 2.3 上下文增强（短追问处理）

```python
# 如果用户只说了"故宫"两个字
if len(prompt) <= 10 and not has_ticket_intent(prompt):
    if recent_spot and recent_spot not in prompt:
        hints.append(f"景区是{recent_spot}")
    # enriched_prompt = "故宫（景区是故宫，用户之前的意图是票务咨询，请基于此回答）"
```

这是解决"用户说短词时 Agent 误判意图"的补丁：
- 上一轮在问票务 → 这一轮说"故宫" → 自动补充意图上下文
- 交给 LLM 时变成完整句子，避免 Agent 转入导览模式

### 2.4 流式输出实现

```python
def capture(generator, cache_list):
    for chunk in generator:
        cache_list.append(chunk)  # 缓存完整回答
        for char in chunk:
            time.sleep(0.01)      # 模拟打字效果（10ms/字）
            yield char

res_stream = agent.execute_stream(enriched_prompt, chat_history)
st.chat_message("assistant").write_stream(capture(res_stream, response_messages))
```

`capture` 做了两件事：
1. 透传 chunk 给 Streamlit 逐字渲染
2. 把完整回答缓存到 `response_messages`，最后存入 `session_state`

### 2.5 历史上下文裁剪

```python
# 只取最近 3 轮（6 条消息）
for msg in st.session_state["message"][-6:]:
    ...
```

为什么只取 3 轮？
- 对话太长 → prompt 超出 token 限制
- 旧轮次的信息已过时（上一个景区的问题跟当前无关）
- 3 轮足够覆盖"追问→回答→追问→回答"的连续场景

## 四、关键代码+逐行注释

### 4.1 app.py 完整核心逻辑
```python
import streamlit as st
from agent.travel_agent import TravelAgent

st.title("文旅智能助手 — 查票 · 订票 · 优惠 · 路线 · 导览")

# ---- 侧边栏：示例查询 ----
with st.sidebar:
    st.subheader("示例查询")
    st.markdown("""
    **查票/订票类：** 帮我查一下故宫明天的余票，3个人
    **票务优惠类：** 两个老人一个儿童去故宫有什么优惠
    **路线规划类：** 帮我规划故宫一日游路线
    **景点导览类：** 午门有什么历史故事
    """)

# ---- 初始化 ----
if "agent" not in st.session_state:
    st.session_state["agent"] = TravelAgent()
if "message" not in st.session_state:
    st.session_state["message"] = []

# ---- 渲染历史消息 ----
for message in st.session_state["message"]:
    avatar = "🧳" if message["role"] == "user" else "🏯"
    st.chat_message(message["role"], avatar=avatar).write(message["content"])

# ---- 用户输入 ----
prompt = st.chat_input()

if prompt:
    # 1) 显示用户消息
    st.chat_message("user", avatar="🧳").write(prompt)
    st.session_state["message"].append({"role": "user", "content": prompt})

    # 2) 构建历史上下文（最近 3 轮）+ 短追问增强
    history_parts = []
    recent_spot = ""
    for msg in st.session_state["message"][-6:]:
        role_label = "用户" if msg["role"] == "user" else "助手"
        history_parts.append(f"{role_label}: {msg['content']}")
        if msg["role"] == "user":
            for spot in ["故宫", "长城", "黄山", ...]:
                if spot in msg["content"]:
                    recent_spot = spot
    chat_history = "\n".join(history_parts)

    # 短追问增强
    enriched_prompt = prompt
    if len(prompt) <= 10 and not has_ticket_intent(prompt):
        if recent_spot and recent_spot not in prompt:
            enriched_prompt = f"{prompt}（景区是{recent_spot}，请基于此回答）"

    # 3) 流式输出
    response_messages = []
    status = st.status("文旅助手思考中...", expanded=False)

    def capture(generator, cache_list):
        for chunk in generator:
            cache_list.append(chunk)
            for char in chunk:
                time.sleep(0.01)
                yield char

    res_stream = st.session_state["agent"].execute_stream(enriched_prompt, chat_history)
    st.chat_message("assistant", avatar="🏯").write_stream(
        capture(res_stream, response_messages)
    )
    status.update(label="已完成", state="complete")

    # 4) 存入历史
    st.session_state["message"].append({
        "role": "assistant",
        "content": response_messages[-1] if response_messages else ""
    })
    st.rerun()  # 刷新页面显示新消息
```

## 六、验收标准+踩坑避坑点

### 6.1 验收标准
- [ ] `python -m streamlit run app.py` 正常启动，浏览器打开 `localhost:8501`
- [ ] 发送"故宫门票多少钱" → 返回阶段 1 概述 + 邀请
- [ ] 发送"两个70岁老人6月15号" → 返回阶段 2 精准计算
- [ ] 历史对话上下文生效：先问票务、再说"故宫"，不误转导览
- [ ] 侧边栏示例点击或手动输入均可正常对话

### 6.2 踩坑避坑点
1. **必须先建向量库**：启动 app 前运行 `python -m rag.vector_store`，否则 `chroma_db/` 不存在导致 Agent 初始化失败
2. **`st.rerun()` 是必需的**：Streamlit 不会自动刷新，追加消息后必须手动 rerun 才能显示新消息
3. **`write_stream` 要求生成器**：`execute_stream` 返回的是生成器，不要 `list()` 化，否则流式效果消失
4. **短追问是补丁不是架构**：上下文增强逻辑在 app.py 而非 Agent 内部，理想情况下应移入 Agent 层统一处理
5. **不要提交 `.env`**：`.gitignore` 必须包含 `.env`，API Key 泄露会被阿里云禁用
